sync.evaluate_candidate(candidate):

        return skip(candidate, "before-sync-baseline")

When The Baseline Is Written

Step Before automation install

Setup should write the baseline after config is available and before LaunchAgent installation, so even an accidental scheduler start sees the cutoff.

Repeat setup Do not move it forward

If sync.ignore_transcripts_before already exists, setup preserves it. Rerunning setup should not silently hide transcripts created since the first setup.

Sync disabled Write when sync is enabled

If the user skips sync automation, no baseline is needed yet. A later setup that enables sync writes the baseline then, if absent.

Implementation Notes

Launchd “Not started” means no immediate execution.

We should be precise: a loaded LaunchAgent with RunAtLoad = false can still participate in future scheduler cycles. Not loading it at all may delay automation until next login.

Baseline Use transcript mtime.

The existing candidate field is modified_at, sourced from Path.stat().st_mtime. That is enough for this slice.

Tests

Setup writes baseline

Run setup with sandboxed home. Assert config contains a sync baseline near test time.

Old transcript skipped

Create a transcript with mtime before baseline. Assert sync status reports before-sync-baseline.

New transcript eligible

Create a transcript after baseline and outside quiet window. Assert it becomes a sync work item.

Out of Scope

No backfill command yet

Do not add sync --backfill in this slice.

No batch-size design yet

Batch limits are a separate product decision. This plan only prevents surprise historical processing.
