# Research Notes

This audit did not require live external research. The relevant prior-art input
was already gathered in the repository and adjacent Almanac work:

- Mature documentation tools use ordinary docs trees and human-readable
  navigation as the primary surface.
- Agent-readable docs still need human-quality prose because agents retrieve
  sections out of context and treat them as authority.
- General Almanac manual discussions argue for articles, leads, citations,
  link context, source limits, and a primary browse tree.
- Wikipedia-style summary and split doctrine supports section READMEs and hubs
  as orientation pages, while keeping detailed subjects in their own articles.

The branch correctly follows those lessons. The remaining work is engineering
cleanup around migration state, not a change in documentation philosophy.
