# Sample Almanac Wiki

This sample wiki documents the non-obvious knowledge a future coding agent should read before changing checkout or payment behavior.

## Notability bar

Write a page when the information would change how a future agent edits this repo:

- A cross-file flow, such as checkout calling payment authorization.
- A decision with rejected alternatives.
- A gotcha discovered through a failed run or production incident.
- A constraint that is not obvious from code alone.

Do not write pages that restate function bodies.
